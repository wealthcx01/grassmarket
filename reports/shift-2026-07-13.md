## e2e wiring — GRS-0019 Playwright specs now run in CI → **PR #21 (MERGED to main)**

The committed Playwright specs were dead weight until CI could execute them. Added an **`e2e`
job** to `.github/workflows/ci.yml`: `npx playwright install --with-deps chromium` (the CI runner
has root, so it installs the system libs the sandbox could not), `scripts/seed_dev.py` seeds the
DB, backend (uvicorn) + built frontend (`next build && next start`) come up with health-waits,
then `npm run e2e`. Traces upload on failure.

- **First CI run caught a real, environment-only bug** — exactly what E2E is for. A bare
  `getByRole("alert")` was a strict-mode violation in a real browser because Next.js injects an
  always-present empty `#__next-route-announcer__` (also `role="alert"`); jsdom never renders that
  announcer, so the component tests were blind to it. Scoped the locator to the refusal text.
- **Result:** all 4 specs pass in CI's stock chromium (generate → review AI sections → approve →
  gate-refusal message). Merged as **PR #21**. E2E now runs on every push.
